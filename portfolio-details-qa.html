QA & Testing – Mobile & Web Application

End-to-end QA for a cross-platform application (web, Android, iOS) before launch. I designed and ran structured test cases, documented bugs and UX issues, and worked closely with developers to improve stability and user experience.

Project Information

  • Category: QA & Testing
  • Platforms: Web, Android, iOS
  • App Type: Mobile and web product before public launch
  • Tools: Google Sheets, Google Docs, device emulators, real devices
  • Role: QA Tester / UX Reviewer

Testing Scope

  • Covered core user flows on web, Android and iOS (onboarding, login, main features, settings).
  • Verified consistency of design and behavior between platforms.
  • Focused on both functional bugs and UI/UX problems that could confuse users.

Responsibilities and Tasks

  • Created structured test cases and checklists for each screen and user flow.
  • Executed tests on different devices and resolutions and documented steps to reproduce issues.
  • Logged bugs with:
    • Screen name and platform (Web, Android, iOS)
    • Steps to reproduce
    • Expected versus actual result
    • Severity and priority
  • Retested fixed issues and updated their status (Open, In progress, Resolved, Verified).

Test Artifacts

  • Google Sheets file for test case management and tracking pass/fail status.
  • Google Docs document with detailed bug reports and screenshots for developers.
  • Organized sections per platform (Web, Android, iOS) to keep communication clear.

Collaboration with Developers and Designers

  • Discussed critical bugs directly with the development team to prioritize fixes before release.
  • Highlighted UI and UX issues such as inconsistent icons, spacing, wording, and navigation, and suggested improvements.
  • Validated each new build, confirming that fixes did not introduce regressions.

Outcome

Thanks to this QA phase, many crashes, navigation glitches, and UI inconsistencies were solved before launch. The app became more stable and clearer to use across web and mobile, resulting in a smoother first release for end users.
